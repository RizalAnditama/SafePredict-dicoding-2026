README Quality Checklist

- [ ] Title and one-line description present
- [ ] TL;DR summarises the project in one sentence
- [ ] Installation steps are copy-paste runnable
- [ ] Usage example included with expected output
- [ ] Testing instructions present and runnable
- [ ] License file present and linked
- [ ] `CONTRIBUTING.md` linked
- [ ] Environment variables documented (`.env.example` provided)
- [ ] CI / build badge present (or placeholder)
- [ ] Links to docs and workplan included

Use this checklist before publishing the repository publicly.
